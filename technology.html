Consensus Protocol

Consensus & Aggregation

No single node controls the outcome. Manipulation requires compromising multiple independent nodes simultaneously — making attacks economically irrational.

1

Random Selection

The protocol randomly selects n nodes from the pool of staked operators. No node knows in advance if it will be chosen, preventing coordinated collusion.

2

Independent Computation

Each selected node independently fetches the same public satellite data (NASA, ESA, NOAA, JAXA), runs the open-source pipeline, and cryptographically signs the result.

3

Aggregation & Consensus

The aggregation smart contract collects all signed reports. Results must agree within a defined tolerance. The consensus value is delivered to the requesting application.

4

Rewards & Slashing

Nodes that reported within consensus earn token rewards. Outlier reporters — whether through error or malice — have their staked tokens slashed. Accuracy pays; manipulation costs.

Why Climate Data Is Uniquely Suited to This Model

Public & Deterministic

Every node accesses the same NASA/ESA/NOAA/JAXA APIs. There is no proprietary data to hide behind — the source data is completely transparent.

Reproducible Computation

Given the same inputs and the same open-source pipeline, every honest node produces the same output. Discrepancies signal either bugs or attacks.

Well-Defined Tolerance

Small floating-point variations are normal and expected. Large deviations are clearly identifiable, making the consensus threshold straightforward to calibrate.

Ground-Truth Validation

Satellite observations can be cross-validated against ground measurements from field sensors and drone data, adding an extra verification layer.

Developer Integration

Start Building in Minutes

Import the ShambaGeoConsumer contract from the npm smart-contract-kit and request verified climate data on-chain.

// SPDX-License-Identifier: MIT
pragma solidity ^0.8.17;

import "@shamba/smart-contract-kit/ShambaGeoConsumer.sol";

contract ClimateInsurance is ShambaGeoConsumer {

    uint256 public latestNDVI;
    uint256 public payoutThreshold;

    constructor(
        uint256 _threshold
    ) {
        payoutThreshold = _threshold;
    }

    function requestNDVI(
        string memory areaOfInterest,
        string memory startDate,
        string memory endDate
    ) public {
        requestGeoData(
            areaOfInterest,
            startDate,
            endDate,
            "ndvi",
            "mean"
        );
    }

    function fulfill(
        bytes32 requestId,
        uint256 result
    ) internal override {
        latestNDVI = result;
        if (result < payoutThreshold) {
            _triggerPayout();
        }
    }
}
